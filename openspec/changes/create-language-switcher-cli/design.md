# Design: Архитектура CLI локализатора

## Architecture
Утилита будет представлять собой Node.js пакет, публикуемый в npm (или используемый локально).

### Компоненты
1. **CLI Entry Point**: Обработка аргументов командной строки (библиотека `commander` или аналог, либо нативный `process.argv`).
2. **Locale Manager**: Модуль для загрузки файлов локализации из папки `locales/`.
3. **File Patcher**: Модуль для чтения и записи `openspec/AGENTS.md`.

### Формат локалей
Локали будут храниться в виде Markdown файлов или JSON объектов, содержащих секции для замены в `AGENTS.md`.
Предлагается хранить полные шаблоны `AGENTS.md` для каждого языка или использовать маркеры для замены частей. Для простоты начнем с полной замены или замены секции инструкций.

### Поток выполнения
1. Пользователь запускает `npx openspec-language-switcher ru`.
2. Утилита проверяет наличие `openspec/AGENTS.md`.
3. Утилита загружает шаблон для `ru`.
4. Утилита перезаписывает `openspec/AGENTS.md` содержимым шаблона `ru`.

## Data Models
- `Locale`: Строка (например, `en`, `ru`).
- `Template`: Содержимое файла Markdown.

## Security
- Утилита работает только с файлами внутри текущей директории проекта.
